Docker Fundamentals

A technical guide to Namespaces, Cgroups, and UnionFS

1. What is Docker?

Docker is a tool that allows you to package an application and its dependencies into a container. Unlike a Virtual Machine, a container does not bundle a full Operating System. Instead, it uses Linux Kernel features to isolate processes from the host system.

2. Linux Namespaces (The Walls)

Namespaces provide isolation by partitioning kernel resources. They create "Filtered Perspectives," where a process believes it has its own private instance of the OS.

  • PID Namespace: Allows a process to have PID 1 inside the container. It cannot see or interact with processes in other namespaces.
  • NET Namespace: Provides a private network stack, including its own IP addresses, routing tables, and firewall rules.
  • MNT Namespace: Isolates mount points, so the container sees a different root (/) filesystem than the host.
  • UTS Namespace: Allows the container to have its own hostname and domain name.
  • USER Namespace: Maps a root user inside the container to a non-privileged user on the host for security.

Technical Implementation: Namespaces are created using the clone() or unshare() system calls with specific flags like CLONE_NEWPID.

3. Control Groups / cgroups (The Rules)

While Namespaces handle what a process can see, Cgroups handle what a process can use. They prevent a single container from consuming all host resources.

  • Resource Limits: Setting maximum thresholds for RAM (e.g., 512MB) and CPU usage.
  • Prioritization: Assigning more CPU "shares" to specific containers.
  • Control: Freezing or restarting groups of processes.

4. UnionFS (The Filesystem)

Union File System is a service that "stacks" multiple directories (branches) to make them appear as one merged directory.

Key Mechanisms:

  • Layering: Docker images are made of read-only layers. These layers are shared across containers to save space.
  • Priority Rule: When two layers contain the same file, the file in the higher-priority (top) layer "shadows" the one below.
  • Copy-on-Write (CoW): If a process tries to modify a read-only file, the Kernel copies it to the top writable layer first. The original remains unchanged.
  • Whiteouts: To "delete" a file from a read-only lower layer, the system creates a "whiteout" file in the writable layer that hides the lower file from the merged view.

5. Comparison: School Setup vs. Docker

Your current environment (at school) uses Standard Permissions. You share the same Namespace, so you can see other users' folders in /home, but "Permission Denied" stops you from entering.

Docker goes further: It uses Namespaces so you cannot even see that other users exist, and UnionFS so that if you "delete" a system file, you only affect your own private "top layer" without touching the actual system.

1. Namespaces  -> Isolation (Walls)
2. Cgroups     -> Restriction (Resource limits)
3. UnionFS     -> Efficiency (Layered Filesystem)
